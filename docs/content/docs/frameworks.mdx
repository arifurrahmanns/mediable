mediable doesn't own routing — it's just functions you call. Every example below is from [the examples/ directory in the repo](https://github.com/arifurrahmanns/mediable/tree/main/examples); each folder has both `addMedia()` server-proxied and `presignUpload()` direct-to-storage flows working out of the box.

All five examples share the same `src/media.ts` config shape — change a few lines and point at any framework.

## Express

Multer handles the multipart. `mediable.addMedia()` accepts `req.file` directly.

```ts
import express from 'express'
import multer from 'multer'
import { media } from './media'

const app = express()
const upload = multer({ storage: multer.memoryStorage() })

app.post('/users/:id/avatar', upload.single('file'), async (req, res) => {
  const record = await media.addMedia({
    model: { type: 'User', id: req.params.id },
    file: req.file!,
    collection: 'avatars',
  })
  res.json(record)
})
```

[Full example →](https://github.com/arifurrahmanns/mediable/tree/main/examples/express)

## Hono

Web-standards `formData()` is built in. No middleware needed.

```ts
import { Hono } from 'hono'
import { media } from './media'

const app = new Hono()

app.post('/users/:id/avatar', async (c) => {
  const form = await c.req.formData()
  const file = form.get('file') as File
  const record = await media.addMedia({
    model: { type: 'User', id: c.req.param('id') },
    file,
    collection: 'avatars',
  })
  return c.json(record)
})

export default app
```

[Full example →](https://github.com/arifurrahmanns/mediable/tree/main/examples/hono)

## Fastify

Laravel-style `req.file()` via `@fastify/multipart`.

```ts
import Fastify from 'fastify'
import multipart from '@fastify/multipart'
import { media } from './media'

const app = Fastify()
await app.register(multipart)

app.post('/users/:id/avatar', async (req, reply) => {
  const data = await req.file()
  if (!data) return reply.code(400).send({ error: 'file required' })
  const record = await media.addMedia({
    model: { type: 'User', id: (req.params as any).id },
    file: { stream: data.file, filename: data.filename, mimetype: data.mimetype },
    collection: 'avatars',
  })
  return record
})
```

[Full example →](https://github.com/arifurrahmanns/mediable/tree/main/examples/fastify)

## NestJS

Decorator-style injection, backed by multer under the hood.

```ts
import { Controller, Post, Param, UploadedFile, UseInterceptors } from '@nestjs/common'
import { FileInterceptor } from '@nestjs/platform-express'
import { media } from './media'

@Controller('users/:id/avatar')
export class AvatarController {
  @Post()
  @UseInterceptors(FileInterceptor('file'))
  async upload(@Param('id') id: string, @UploadedFile() file: Express.Multer.File) {
    return media.addMedia({ model: { type: 'User', id }, file, collection: 'avatars' })
  }
}
```

[Full example →](https://github.com/arifurrahmanns/mediable/tree/main/examples/nestjs)

## Next.js App Router

Fetch-style route handlers. Pin `runtime = 'nodejs'` — mediable uses node:fs, sharp, pg.

```ts
// app/api/users/[id]/avatar/route.ts
import { media } from '@/lib/media'

export const runtime = 'nodejs'

export async function POST(req: Request, { params }: { params: Promise<{ id: string }> }) {
  const { id } = await params
  const form = await req.formData()
  const file = form.get('file') as File
  const record = await media.addMedia({
    model: { type: 'User', id },
    file,
    collection: 'avatars',
  })
  return Response.json(record)
}
```

## Direct-to-storage in every framework

Every example ships both the multipart route and the `/api/presign-upload` + `/api/confirm-upload` routes. Set `S3_BUCKET` + `S3_REGION` + `S3_ACCESS_KEY` + `S3_SECRET_KEY` (and `S3_ENDPOINT` for R2/MinIO/B2) in `.env` and the direct-upload flow lights up automatically.

See [Direct-to-storage uploads](/docs/direct-upload) for the full flow and frontend code.
